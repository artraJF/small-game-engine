# small-game-engine
Using this repo as a place to practice C/C++ while very slowly creating a 3D game engine.

From what I skimmed online, some people do not recommend using a list of wanted features, but rather, create an engine for the game you want to make, and then add features according to what you need.
Now, I do not know the first thing about making a game engine, and this is more of a fun personal project to create a generic engine, so I'm going to forgoe this advice and make a list anyway. It will be more of a loose set of tasks that I would like to tackle, for now.

So without further ado:
* Create a window
    * and subsequent buttons for each added feature
* Render a 3D object
    * Read and add a 3d object from an external file
* Add collision
    * and basic physics
* Add a sound engine
* Add a camera
* Add lighting (very optional for now)
* Add optimisation
    * Frustum culling and watnot

There are obviously way more features to add here (more low level stuff like multi-threading for example), but I'll keep this high-level list for now and proceed from here.
